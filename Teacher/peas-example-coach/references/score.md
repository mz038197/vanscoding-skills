## 總分（0～100）— Sigmoid 換算

設五軸分數加總為 **S**（每軸 1～5，故 **5 ≤ S ≤ 25**）。總分改以 **logistic（sigmoid）** 做非線性壓縮後，再**線性拉伸**到 **0～100**，使 **S = 5 必為 0 分、S = 25 必為 100 分**（端點與線性換算相同，中間分數分佈較「中間略緩、往兩端變陡」，可依 **k** 調整）。

### 符號

- **t** = (S − 15) ÷ 5 — 將 S 對齊到以 **15**（五軸皆 3 時的加總）為中心，且 **S = 5 → t = −2**、**S = 25 → t = +2**
- **k** — 陡峭係數，**預設 k = 2**（**k 越大**，同一 S 變化在總分上對應的落差越集中在「接近很好／很差」的區段）
- **σ(z)** = 1 ÷ (1 + e^(−z)) — 標準 sigmoid
- **z** = **k × t**

### 公式

1. **z** = k × (S − 15) ÷ 5  
2. **σ_lo** = σ(−2k)、**σ_hi** = σ(+2k) — 對應 S = 5 與 S = 25 時的 σ 值  
3. **總分** = **round( 100 × (σ(z) − σ_lo) ÷ (σ_hi − σ_lo) )**（四捨五入為整數）

**邊界**：S = 5 → z = −2k → σ(z) = σ_lo → **0**；S = 25 → z = 2k → σ(z) = σ_hi → **100**。

### 範例（k = 2）

概念 4、流程 5、實驗 4、表達 4、自主 4 → **S = 21** → t = 1.2 → z = 2.4 → **總分約 93**（線性舊算法同例為 80；改 sigmoid 後同加總會落在較高分段，屬預期行為）。

若需較接近線性的手感，可改 **k = 1**（仍為 sigmoid，曲線較平緩）。

產出時請在摘要末行（或五軸列表下一行）明確標示，例如：

**總分（0–100）：93**（Sigmoid，k = 2）

## 面向說明（供評分時對齊）

| 面向 | 涵義（簡要） |
|------|----------------|
| 概念理解 | 是否掌握名詞、因果與「為什麼」，能對照程式或教材說清楚。 |
| 流程理解 | 是否掌握步驟順序、資料／訊息如何流動、關鍵分支與時機。 |
| 實驗與證據 | 能否以操作、輸出、行號或觀察結果支撐說法，而非僅背誦。 |
| 表達清晰度 | 用自己的話是否條理清楚、與 checklist「能指到對應位置」一致。 |
| 自主性 | 在提示與追問下，能否自行推進與修正，而非長期依賴直接給答案。 |
