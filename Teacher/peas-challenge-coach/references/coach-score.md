# 理解評估摘要

**與 peas-example-coach 格式完全相同**：同標題、同五軸（各 1–5）、同 Sigmoid 總分公式，避免學生在範例陪練與進階挑戰兩階段看到不同評分架構。

教練模式**階段 6**（**必做** Challenge 全部完成後）產出時使用本格式。產出後除對話呈現外，須將全文寫入專案 **`session-records/peas-challenge-score.md`**（或與工作階段紀錄對齊之日期後綴檔名），見 skill 主檔「輸入」第 3 點。評分時對照工作階段紀錄中的**實作紀錄**與驗收對談內容（**僅必做** Challenge）。**五個面向各給 1–5 分**（整數刻度與加總 **S**、Sigmoid 公式不變）。**對話與落檔時，五軸分數以星星為主視覺**（見下「星星對照」），並在**同一行括號內註明（X／5）**供助教核對與代入公式。**五軸填完後**，須再依下方公式計算並寫出 **總分（0–100）**；選修見下「選修／挑戰加分」。整體評估仍以五軸文字為主，總分為濃縮參考，**不可**只報總分而省略五軸與說明。

## 總分（0～100）— Sigmoid 換算

設五軸分數加總為 **S**（每軸 1～5，故 **5 ≤ S ≤ 25**）。總分改以 **logistic（sigmoid）** 做非線性壓縮後，再**線性拉伸**到 **0～100**，使 **S = 5 必為 0 分、S = 25 必為 100 分**（端點與線性換算相同，中間分數分佈較「中間略緩、往兩端變陡」，可依 **k** 調整）。

### 符號

- **t** = (S − 15) ÷ 5 — 將 S 對齊到以 **15**（五軸皆 3 時的加總）為中心，且 **S = 5 → t = −2**、**S = 25 → t = +2**
- **k** — 陡峭係數，**預設 k = 2**（**k 越大**，同一 S 變化在總分上對應的落差越集中在「接近很好／很差」的區段）
- **σ(z)** = 1 ÷ (1 + e^(−z)) — 標準 sigmoid
- **z** = **k × t**

### 公式

1. **z** = k × (S − 15) ÷ 5  
2. **σ_lo** = σ(−2k)、**σ_hi** = σ(+2k) — 對應 S = 5 與 S = 25 時的 σ 值  
3. **總分** = **round( 100 × (σ(z) − σ_lo) ÷ (σ_hi − σ_lo) )**（四捨五入為整數）

**邊界**：S = 5 → z = −2k → σ(z) = σ_lo → **0**；S = 25 → z = 2k → σ(z) = σ_hi → **100**。

### 範例（k = 2）

概念 4、流程 5、實驗 4、表達 4、自主 4 → **S = 21** → t = 1.2 → z = 2.4 → **總分約 93**（線性舊算法同例為 80；改 sigmoid 後同加總會落在較高分段，屬預期行為）。

若需較接近線性的手感，可改 **k = 1**（仍為 sigmoid，曲線較平緩）。

產出時請在摘要末行（或五軸列表下一行）明確標示，例如：

**總分（0–100）：93**（Sigmoid，k = 2）

## 選修／挑戰加分（不與正式總分混算）

- **正式計分範圍**：**五軸**與 **S → 總分（0～100）** 僅依**必做** Challenge 之驗收對談與實作紀錄取證（見 peas-challenge-coach「進度顯示」之 **N**：不含標記為選修之 Challenge）。
- **選修 Challenge**：**不納入**五軸敘述之主要依據，**不**因完成與否調整 **S** 或 Sigmoid 總分。
- **挑戰加分（選修）**：學生若完成選修 Challenge，可於**理解評估摘要**中、**總分一行之後**另起一小節標題 `### 挑戰加分（選修）`，以 1～3 句自然語簡列完成了哪些額外挑戰、一句肯定或觀察（**不**另給數字分數、**不**併入 S）。若未做選修，可省略此小節或寫「未挑戰選修項目」。

## 面向說明（供評分時對齊）

以下涵義**同時**適用 peas-example-coach（陪練／思考格）與 peas-challenge-coach（實作／驗收）；教練階段請特別從**程式行為、驗收對談、實作紀錄**取證。

| 面向 | 涵義（簡要） |
|------|----------------|
| 概念理解 | 是否掌握名詞、因果與「為什麼」，能對照程式或教材說清楚。進階實作時包含能否說明設計與規格背後的理由，而非僅「照做」。 |
| 流程理解 | 是否掌握步驟順序、資料／訊息如何流動、關鍵分支與時機（含讀檔、寫檔、回合邊界等，若題目有涉）。 |
| 實驗與證據 | 能否以操作、終端輸出、行號或觀察結果支撐說法，而非僅背誦；進階實作含跑程式驗收、錯誤訊息、除錯嘗試等可核對線索。 |
| 表達清晰度 | 用自己的話是否條理清楚、與「能指到對應位置」（程式、檔案或決策）一致。 |
| 自主性 | 在提示與追問下，能否自行推進與修正，而非長期依賴直接給答案或逐步代做。 |
