# 實作紀錄模板（每題一則，寫入 md）

與思考格互補；思考格記錄「理解了什麼」，實作紀錄記錄「做了什麼、為什麼這樣做」。

---

### Challenge X：<Challenge 標題>

- **題意摘要**：用一句話描述這題要做什麼（由 agent 根據 challenges.md 填寫，不直接複製規格原文）。
- **實作方式**：學生採取的做法概述 — 新增了哪些函式／區塊、改了哪幾行、用了什麼資料結構。若學生的做法與 challenges.md 建議不同，在此註明「替代方案」並簡述差異。
- **遇到的問題**：實作過程中卡關的地方、錯誤訊息、除錯過程（學生自述或 agent 觀察）。若一路順利可寫「無明顯卡關」。
- **設計決策 / 理解**：學生對驗收條件中「能說明…」題目的回答摘要（由學生口述，agent 整理）。例如：為什麼不存 SystemMessage、為什麼要整輪刪除、cursor 與 history 的對應關係。
- **驗收結果**：
  - [ ] <驗收條件 1 精簡描述> — ✅ 通過 / ❌ 未通過（附註）
  - [ ] <驗收條件 2 精簡描述> — ✅ 通過 / ❌ 未通過（附註）
  - …（依該題驗收條件數量展開）
- **Agent 備註**：值得注意的學習亮點、仍可深入的方向、或建議下次複習的概念（一兩句即可）。
